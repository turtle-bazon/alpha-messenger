# 28. Явные дефолты уведомлений и нормализация denied (задача #29)

Починен рассинхрон настроек уведомлений после очистки localStorage. Раньше отсутствие ключа неявно значило «включено», поэтому после очистки ключей не было вовсе, а при заблокированном системном разрешении возникало расхождение «логически browser=вкл, но в UI выкл». Добавлена `initNotifDefaults()` (`util/notifications.ts`), вызывается из `HomeScreen` при входе до `ensureBrowserPermission()`: сидит `sound='1'` при отсутствии ключа; для `browser` при `permission==='denied'` принудительно пишет `'0'` (и больше не включает/не спрашивает), иначе при отсутствии ключа — `'1'`. Явный пользовательский выбор не перетирается, кроме принудительного `'0'` при denied. Покрыто `e2e/notif-defaults.spec.ts` (granted→оба `'1'`; denied→`browser='0'` + выключенный недоступный тумблер с подсказкой). Регресс 31/31.
