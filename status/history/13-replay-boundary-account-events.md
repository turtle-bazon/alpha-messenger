# 13. Граница реплея и события аккаунта

В поток событий введена граница реплея: сервер после hello-выгрузки истории шлёт управляющий маркер `synced` (без `seq`), клиент по нему отделяет «уже было» от live (WsClient.isLive()). На этом построены уведомления уровня аккаунта в UI (новый вход/новое устройство по auth.attempt+device.added, тост с авто-скрытием, свои события — по deviceId) и устранено задвоение счётчиков непрочитанного на холодном старте. Закреплён принцип: состояние — из REST, события — для live и догона пропуска. Покрыто e2e/account-events.spec.ts.
