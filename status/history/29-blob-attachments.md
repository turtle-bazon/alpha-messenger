# 29. Вложения через блобы: хранилище и изображения (#31, #17)

Крупные вложения вынесены из тела сообщения в отдельное content-addressed хранилище. Сервер хранит непрозрачные блоба, адресуемые по sha256 содержимого (= blobId), и не знает ни содержимого, ни ключей (в перспективе блоб шифруется клиентом, как и сообщения).

Сервер (#31, коммит d9b4729). Миграция `0005_blobs.sql`: `blobs` (blob_id, size), `blob_owners` (загрузчики; у дедуплицированного блоба их несколько), `message_blobs` (связь сообщение↔блоб). `POST /api/blobs` — стрим сырого тела (octet-stream), sha256 на лету, лимит `MAX_BLOB_SIZE` (413), дедуп. `GET /api/blobs/{id}` — доступ загрузчику **или** участнику чата с неудалённым сообщением, ссылающимся на блоб. У сообщений — открытое поле `blobIds[]` (из ciphertext сервер ссылки не прочтёт, а они нужны для авторизации и refcount). Абстракция `BlobStore` с драйверами `fs` (dev/тесты) и `s3`/MinIO (prod/deploy), выбор по env `BLOB_STORE`. Тест `blobs.test.ts`.

Клиент — изображения (#31 этап 2). Модель контента → `{ text, attachments[] }` (тело `t:'msg'`), вложение `ImageAttachment` с `blobId`, размерами, тонким inline-thumbnail и зарезервированным `key` под E2EE; чтение легаси (`t:'image'`/`t:'text'`) сохранено. Картинка едет по модели «thumbnail inline + полноразмер в блобе» (как у Telegram): `prepareImage()` делает полноразмерный JPEG-блоб (≤2560px) и крошечный thumbnail (≤320px); отправка через очередь `pump()` сперва грузит блоб (`uploadBlob`), затем шлёт `blobIds` (порядок/ретрай сохранены). В пузыре — thumbnail, по клику — полноэкранный просмотр (`MediaViewer`), полный файл тянется `fetch`+токеном в object-URL (кеш `blobUrl.ts`). `Message.blobIds` на клиент не добавляли — blobId читается из расшифрованного контента.

Вставка из буфера (#17). `onPaste` на поле ввода: картинка из буфера открывает тот же `ImageEditor`, что и 📎, и едет тем же блоб-путём; при редактировании сообщения отключено.

Покрытие: e2e `image.spec.ts` (доставка thumbnail по WS, открытие полноразмера из блоба у обоих участников, вставка из буфера). Регресс клиента 33/33, серверные тесты 14/14. Follow-up — в issue #31 (файлы, видео+range, drag&drop, мультивложения, GC/refcount, presigned URL, шифрование блоба).
