# 31. Статусы присутствия/тайпинга в ChatList и устойчивость превью

Две задачи в одном заходе по списку чатов.

## #27 — индикаторы на аватаре (присутствие/тайпинг)

Единый источник «кто печатает» — хук `chats/useTyping.ts` (`chatId → Set<userId>` по транзиентным WS-событиям `typing`), поднят в `HomeScreen` и кормит список чатов, заголовок переписки и окно участников. Тайпинг держится 6 с и снимается досрочно по `message.new` отправителя (как в Telegram) и по `presence: offline`. Локальную подписку на `typing` из `Conversation` убрали — заголовок читает проп `typingUsers`.

Визуал — компонент `chats/AvatarBadges.tsx` + стили: кружок присутствия снизу-справа (зелёный/серый, только личные чаты; у групп нет) и пульсирующая окантовка «печатает» (личные — собеседник; группы — любой участник, в `ChatList` и `MembersDialog`). Присутствие берётся из имевшегося `onlineUsers`, проброшенного в `ChatList`.

Сопутствующе: при отправке сбрасываем троттл `typing`, чтобы следующий ввод сразу уведомил собеседника. Сценарий — `e2e/presence-status.spec.ts`, хелпер `createGroupViaUi`.

## #28 — превью последнего сообщения (баг невоспроизводим)

Гипотеза задачи («инкрементально через WS lastMessage не обновляется») не подтвердилась: обработчик `message.new` в `HomeScreen` обновляет превью во всех типичных сценариях (приём при открытом/закрытом чате, своё сообщение, оффлайн+вход, reconnect, картинка). Нашли и поправили единственную реальную гонку — ветка `idx < 0`: на всплеск сообщений во вновь появившийся чат летят параллельные `getChat`, и превью могло застрять на раннем сообщении; теперь берём самое свежее (`newestPreview`) из снимка, элемента списка и события. Регресс — `e2e/preview-stale.spec.ts`. Задача закрыта с пометкой «не воспроизводится, поправили найденное».
