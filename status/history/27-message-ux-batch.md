# 27. Пакет UX переписки (задачи #19, #21, #24, #25, #26)

Пять задач по переписке в web-клиенте (всё в Conversation.tsx, icons.tsx, index.css; покрыто новыми e2e composer.spec.ts, group-identity.spec.ts, retry.spec.ts + перевод interactions/two-clients на data-status). (#19) Кнопка отправки всегда видна на стабильной позиции, при пустом поле — погашена (disabled), без анимации появления. (#25) Поле ввода переведено на textarea с авторасширением под содержимое (до 160px, дальше скролл); Enter — отправка, Shift+Enter — перенос строки. (#24) Галочки статуса — inline-SVG вместо текстовых ✓: отправлено (одиночная серая), прочитано (двойная синяя), плюс спиннер «отправка» и «!» при ошибке; на элемент добавлен data-status. Отдельного «доставлено» нет — сервер не различает доставку и отправку. (#21) Идентификация автора в группе: имя над первым пузырём серии (цвет по автору) и аватар у последнего; в личных чатах и у своих сообщений — ничего. (#26) Последовательная очередь отправки (pump): строгий порядок, при ошибке голова блокирует очередь (позднее не обгоняет раннее); failed-сообщение с кнопкой «Повторить»; автоповтор по window online, WS-маркеру synced и по таймеру. Регресс e2e 29/29.
