# 21. Добавление участников в группу

Добавление участников в группу создателем (как в Telegram; удаление уже было — п.19). Эндпоинт POST /api/chats/{id}/members с телом { username }: право только у создателя, лишь для группы, нельзя добавить уже состоящего (409). Новому участнику пишется chat.created (подтягивает чат в список), уже состоящим — новое событие chat.member_added (обновляют состав/счётчик участников; HomeScreen перечитывает чат из REST и пересеивает presence). Клиент: в MembersDialog у создателя появилась форма «Добавить по username» (под списком). Покрыто серверным test/members.test.ts (право/группа/дубль/несуществующий) и e2e members.spec.ts (двухконтекстный сценарий: создатель добавляет живого клиента — у него чат появляется вживую, у создателя растёт счётчик; не-создатель формы не видит). Регресс: сервер 11/11, e2e 15/15.
