# 25. Починка задвоения сообщений

Починено задвоение сообщений в открытой переписке (рандомно, после переключения чатов; дублировались и входящие, и исходящие целым блоком). Корень — единственный путь добавления без дедупликации: ленивая подгрузка истории onScroll сливала страницу через push (остальные пути идут через upsert по messageId). Перекрытие диапазонов запускалось повторным входом в onScroll (guard `loadingMore` жил в React-state) и протеканием hasMore/nextBefore прошлого чата (Conversation без key={chatId}). Решение: key={chatId} на Conversation (свежий маунт на каждый чат), guard повторного входа на ref, дедуп слияния страницы по messageId. Покрыто новым e2e message-dedup.spec.ts; регресс e2e 21/21.
