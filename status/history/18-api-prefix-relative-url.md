# 18. Префикс /api/ и относительные URL за прокси

Все серверные REST-эндпоинты объединены под префиксом /api/ (на сервере — плагин с prefix:'/api', /ws остаётся в корне), что упростило обратный Apache-прокси до одного правила ProxyPass /api/. Заодно устранена привязка клиента к localhost:3000 за прокси: в прод-сборке VITE_API_URL='' даёт относительные URL (запросы на тот же origin — CORS не возникает, образ переносим), wsUrl() при пустом адресе берёт схему/хост из window.location; build-arg проброшен пустым в CI, а в client Dockerfile дефолт localhost:3000 сохраняет локальную run/prod-сборку. Покрыто прогоном серверных тестов 8/8 и e2e-регрессом 9/9.
