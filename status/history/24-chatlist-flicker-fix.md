# 24. Починка мигания списка чатов на логине

Починено «мигание» списка чатов на логине (known_issues №4): реплей outbox перерисовывал список десятки раз и дёргал getChat/getPresence на каждый чат. WsClient теперь буферизует реплей и применяет его одним пакетом на маркер 'synced' (один ререндер; live включается после флаша, поэтому реплей по-прежнему не путается с live). HomeScreen коннектит WS только после загрузки списка из getChats (реплей ложится на готовый список), а точечные getChat/getPresence в обработчиках chat.created/member_added/member_removed зовутся лишь для реально отсутствующего чата или только вживую (на реплее состояние уже из REST + единый снимок presence по 'synced'). Закреплён принцип: на холодном старте состояние списка — из REST, реплей лишь догоняет, не переинициализирует. Покрыто новым e2e login-replay.spec.ts; регресс e2e 20/20.
