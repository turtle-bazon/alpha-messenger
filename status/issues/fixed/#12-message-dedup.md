# #12 — Задвоение сообщений в открытой переписке

[Баг] Сообщения задваивались в открытой переписке — рандомно, после переключения чатов (дублировались и отправленные, и полученные, целым блоком). Причина: дубль с одинаковым messageId мог породить только один путь — ленивая подгрузка истории по скроллу вверх (onScroll), где страница сливалась в список через `merged.push(...)` БЕЗ дедупликации (все остальные пути идут через upsert, дедуп по messageId). Перекрытие диапазонов возникало двумя способами: (а) защита от повторного входа `loadingMore` жила в React-state — несколько scroll-событий в одном тике читали старое (false) значение, проскакивали guard и грузили одну и ту же страницу дважды; (б) Conversation монтировался без key={chatId} — при переключении чатов инстанс переиспользовался, и пока новый чат грузился, hasMore/nextBefore ещё держали значения прошлого чата, из-за чего onScroll по пустому списку (scrollTop≈0) подгружал текущий чат по чужому nextBefore. Решение: (1) key={chatId} на Conversation в HomeScreen — переключение чата даёт свежий маунт (hasMore=false, nextBefore=null), стейт прошлого чата не протекает; (2) guard повторного входа переведён на ref (loadingMoreRef) — синхронно отсекает параллельные scroll-события; (3) слияние страницы дедуплицирует по messageId (страховка от любого перекрытия диапазонов). Покрыто новым e2e message-dedup.spec.ts (60 сообщений, два scroll-события в одном тике → ровно 60 без дублей; падает на старом коде, проходит на новом). Регресс e2e 21/21.
