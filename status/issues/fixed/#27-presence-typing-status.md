# #27 — Статусы онлайн/оффлайн/тайпинг в ChatList

Статусы онлайн/оффлайн/тайпинг в ChatList как в Telegram. Реализовать три состояния аватара:
- **Онлайн**: зелёный кружок внизу справа аватара (12-14px). Только для DM.
- **Оффлайн**: серый кружок внизу справа аватара. Только для DM.
- **Тайпинг**: большой круг-окантовка вокруг всего аватара (2-3px border). Цвет — контрастный с аватаром (пусть клод подберёт подходящий). Появляется: в DM — если собеседник печатает тебе; в групповых чатах — не показывать (или показывать в списке участников при необходимости).
- **Групповые чаты**: кружок статуса (онлайн/оффлайн) НЕ показывать. Тайпинг-окантовку показывать ВЕЗДЕ где есть списки участников в групповых чатах (ChatList, MembersDialog, и т.д.) — если участник печатает в этот чат, обернуть его аватар в тайпинг-окантовку.

Компонент: позиционирование bottom:0 right:0 для кружков, border-radius:50% для тайпинга. Не путать с AvatarStatus (оранжевый «typing» кружок) — это два разных индикатора.

## Решение

Единый источник состояния «кто печатает» — хук `chats/useTyping.ts`: `chatId → Set<userId>` по транзиентным WS-событиям `typing`. Тайпинг каждого пользователя держится 6 с после последнего события (как раньше в Conversation) и снимается досрочно, если: пришло его `message.new` (набор завершён отправкой — как в Telegram) либо его `presence: offline`. Хук поднят в `HomeScreen` и кормит сразу список чатов, заголовок переписки и окно участников — без дублирующих подписок (локальную подписку на `typing` из `Conversation` убрали, заголовок теперь читает проп `typingUsers`).

Индикаторы на аватаре — компонент `chats/AvatarBadges.tsx` (кружок присутствия + окантовка) и стили в `index.css`:
* **Присутствие** (`.avatar-status-dot`, 13px, bottom/right): зелёный — онлайн, серый — оффлайн. Только для личных чатов; у групп кружок не показываем.
* **Тайпинг** (`.avatar-typing-ring`, окантовка 2px акцентным цветом, `border-radius:50%`, пульсация): в личных — если печатает собеседник; в группах — если печатает любой участник (на аватаре чата в списке и на аватаре конкретного участника в `MembersDialog`).

Источник присутствия — уже имевшееся множество `onlineUsers` (снимок `GET /presence` + живые события `presence`), проброшено в `ChatList`.

Доп. правка UX: при отправке сообщения сбрасываем троттл `typing` (`lastTypingSent`), чтобы следующий ввод сразу заново уведомил собеседника.

Сценарии: `e2e/presence-status.spec.ts` — кружок онлайн→оффлайн в личном чате, окантовка «печатает» в личном (появляется при наборе, гаснет после отправки), у группы нет кружка, но есть окантовка при наборе участника. Хелпер `createGroupViaUi` добавлен в `e2e/helpers/ui.ts`. Регресс заголовка «печатает» — `interactions.spec.ts` (зелёный).
