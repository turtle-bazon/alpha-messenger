# #53 — Кнопка «назад» после перехода к сообщению

После перехода к сообщению (клик по reply preview) — показывать кнопку «назад», которая возвращает к предыдущей позиции скролла.

## Поведение
- Клик по reply preview → скролл к оригиналу + подсветка
- В правом верхнем углу области переписки появляется кнопка «↩» (или стрелка вниз)
- Клик по кнопке → скролл обратно к сообщению, на котороекликали (к reply preview)
- Кнопка исчезает после возврата

## Как в Telegram
- В Telegram при переходе по ответу внизу справа появляется круглая кнопка со стрелкой
- Нажатие возвращает к исходному сообщению

## Реализация
- Сохранять `scrollTo` (ID сообщения, откуда перешли) при клике по reply preview
- Если `scrollTo !== null` — показывать кнопку
- При клике по кнопке → scrollIntoView к `scrollTo`, сбросить `scrollTo`

## Приоритет
normal

## Решение
Реализовано в `Conversation.tsx`:
- `navStackRef` — стек сохранённых `scrollTop` позиций
- `pushNavStack()` — сохраняет текущую позицию скролла перед прыжком
- `goBack()` — восстанавливает предыдущую позицию из стека
- `showBackBtn` — управляет видимостью кнопки `nav-back`
- Клик по `.bubble-reply` вызывает `pushNavStack()` + `scrollIntoView` + подсветку
- CSS: `.nav-back-btn` — круглая кнопка в правом верхнем углу с анимацией fadeInUp
