# #13 — Уведомления о непрочитанных сообщениях

Уведомления о непрочитанных сообщениях в web-клиенте (три механизма, по умолчанию всё включено): (а) браузерное системное уведомление при входящем; (б) счётчик непрочитанных в title вкладки («(3) alpha»); (в) короткий звук. Звук и попап срабатывают, только когда вкладка не активна (document.hidden или потерян фокус — как в Telegram Web); при активной вкладке хватает badge в списке и счётчика в title. Реализация: util/notifications.ts (настройки sound/browser в localStorage с дефолтом «включено», запрос Notification.permission, playSound через WebAudio без ассета, setUnreadBadge для title, notifyIncoming с проверкой видимости вкладки), компонент NotificationSettings (колокольчик в шапке + выпадающее меню с двумя тумблерами; включение браузерных запрашивает системное разрешение, при denied тумблер заблокирован с подсказкой), иконки IconBell/IconBellOff. Браузерные попапы работают «из коробки»: приложение само запрашивает Notification.permission при входе (ensureBrowserPermission в HomeScreen), если настройка включена, а разрешение ещё 'default' — иначе попапы молча не показывались бы (permission по умолчанию не granted), а тумблер прятал бы запрос. Тумблер браузерных честно отражает browser && permission==='granted'. В HomeScreen: вызов notifyIncoming в обработчике message.new (только живое чужое сообщение, имя чата из chatsRef) и эффект суммы unreadCount → setUnreadBadge. Отсекаются собственные и реплейные события (как у security-уведомлений). Покрыто новым e2e notifications.spec.ts (badge в title + перехват браузерного уведомления при неактивной вкладке через подмену Notification; переключение и сохранение настроек между сессиями). Регресс e2e 23/23.
