# #11 — Мигание списка чатов при логине

[Баг] Список чатов «мигал»/многократно перерисовывался при логине. Причины: (а) сервер шлёт каждое replay-событие отдельным WS-кадром → отдельный onmessage → отдельный setChats → отдельный ререндер (на холодном старте с большой историей — десятки перерисовок); (б) ws.connect() стартовал параллельно с getChats(), и replay-обработчики chat.created/message.new дёргали getChat/getPresence на каждый чат, хотя список уже авторитетен из getChats. Решение: (1) WsClient буферизует все события реплея и применяет их одним синхронным пакетом на маркер 'synced' (React 18 батчит в один ререндер; live включается после флаша, поэтому реплей по-прежнему не считается за live); (2) HomeScreen коннектит WS только после resolve getChats() — реплей ложится на готовый список; (3) точечные getChat/getPresence в обработчиках chat.created/member_added/member_removed вызываются только для реально отсутствующего чата либо только вживую (ws.isLive()), на реплее состав/presence уже из getChats + единого снимка по 'synced'. Покрыто новым e2e login-replay.spec.ts (после повторного входа с lastSeq=0 точечный getChat не вызывается ни разу, список восстановлен из getChats). Регресс e2e 20/20.
