# #17 — Вставка изображений из буфера обмена

Поддержка вставки изображений из буфера обмена (Ctrl+V / Cmd+V) в поле ввода сообщения в Conversation. При вставке картинки — открывать ImageEditor (как при прикреплении через 📎) для предпросмотра, поворота, подписи и отправки.

## Решение
Обработчик `onPaste` на textarea композера (`chats/Conversation.tsx`): перебирает
`clipboardData.items`, и если есть элемент `kind === 'file'` с `type`
`image/*` — берёт файл, `preventDefault()` (чтобы в поле не вставился
сопутствующий текст) и кладёт его в `pendingImage`. Дальше — тот же путь, что и
при прикреплении через 📎: открывается `ImageEditor`, а отправка идёт блоб-путём
(полный файл в блоб + thumbnail inline, см. #31). Во время редактирования
сообщения вставка картинки отключена — работает обычная вставка текста.

Тест `e2e/image.spec.ts` — «вставка изображения из буфера…»: синтезирует real-
событие `paste` с `clipboardData`, содержащим image-файл (clipboardData задаётся
через `Object.defineProperty` — конструктор события его не выставляет), проверяет
открытие редактора, отправку и доставку картинки собеседнику по WS.
