# #41 — Синхронизация черновика между девайсами

## Описание
Текст в поле ввода (черновик) должен синхронизироваться между устройствами. Если пользователь начал печатать на одном устройстве, он может продолжить на другом.

## Требования
1. **Сервер хранит черновик** — при изменении текста в поле ввода отправлять его на сервер (debounce ~1-2 сек)
2. **Черновик привязан к чату** — каждый чат имеет свой черновик
3. **Загрузка при открытии чата** — при выборе чата загружать черновик с сервера
4. **Очистка при отправке** — после отправки сообщения черновик удаляется

## Реализация

### Сервер
- Добавить таблицу `drafts` (chat_id, user_id, ciphertext, updated_at)
- Эндпоинт `PUT /api/chats/:chatId/draft` — сохранить черновик
- Эндпоинт `GET /api/chats/:chatId/draft` — получить черновик
- Эндпоинт `DELETE /api/chats/:chatId/draft` — удалить черновик (при отправке)

### Клиент
- Отслеживать изменение текста в `Conversation.tsx` с debounce
- При открытии чата загружать черновик с сервера
- При отправке сообщения удалять черновик
- При переключении чатов сохранять текущий черновик

### API
```
PUT /api/chats/:chatId/draft
Body: { ciphertext: string }

GET /api/chats/:chatId/draft
Response: { ciphertext: string } | null

DELETE /api/chats/:chatId/draft
```

## Приоритет
Minor — полезная фича, но не критичная

## Статус
Открыта

## Решение
- Сервер: таблица `drafts` (PK: chat_id + user_id), эндпоинты GET/PUT/DELETE `/api/chats/:chatId/draft`
- Клиент: `getDraft()`, `saveDraft()`, `deleteDraft()` в `rest.ts`
- `Conversation.tsx`: загрузка черновика при смене `chatId`, сохранение с debounce 1.5с, удаление при отправке
- E2E тест: `draft-sync.spec.ts` — восстановление после перезагрузки
