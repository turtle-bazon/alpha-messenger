# #36 — Статус «отошёл» и «последний раз был активен»

Реализовать статус «отошёл» для пользователей, которые залогинены, но давно не проявляли активность в мессенджере.

## Серверная часть

### Отслеживание активности
- Новая колонка в `accounts`: `last_active_at` (timestamptz, nullable). Обновляется при любой активности пользователя.
- Обновление `last_active_at`:
  - При отправке сообщения (POST /api/messages) — уже есть.
  - При WS-соединении / reconnect — обновлять при hello.
  - Новый REST-эндпоинт: `POST /api/me/activity` — «пинг» активности. Клиент шлёт при обнаружении движения мыши, кликах, нажатиях клавиш (throttle ~30 сек).
- В `loadChat` и `GET /api/presence` добавить `lastActiveAt` для каждого участника.

### Определение статуса
- `online` — есть живой WS-connection (как сейчас).
- `away` — `last_active_at` > 5 минут назад, но WS есть.
- `offline` — нет WS-connection.

## Клиентская часть

### Отправка пингов активности
- В `HomeScreen` или `App` — обработчик `mousemove`, `keydown`, `focus` с throttle 30 сек.
- При срабатывании — `POST /api/me/activity` (fire-and-forget, без ожидания ответа).
- Не шлём пинг если вкладка не активна (`document.hidden`).

### Отображение статуса в ChatList
- Маленький кружок статуса (уже есть: зелёный online, серый offline) — расширить на три состояния:
  - `online` — зелёный кружок (как сейчас).
  - `away` — жёлтый/оранжевый кружок.
  - `offline` — серый кружок (как сейчас).
- Всё остальное в ChatList без изменений (имя, последнее сообщение, время).

### Отображение в заголовке переписки (DM)
- Под именем собеседника — строка с информацией о последней активности:
  - `online` — «в сети» (зелёный)
  - `away` — «отошёл. Был(а) X мин. назад» (жёлтый)
  - `offline` — «Был(а) X [мин/ч/дн/нед] назад (с ДД.ММ.ГГГГ)» (серый)
- Шрифт: 13px, `var(--text-muted)`, отступ от имени 2px.
- Для групп: «N участников, M в сети» (без change).

### Отображение в списке участников (MembersDialog)
- Под именем каждого участника — нижняя строка с информацией о последней активности:
  - `online` — «в сети» (зелёный)
  - `away` — «отошёл. Был(а) X мин. назад» (жёлтый)
  - `offline` — «Был(а) X [мин/ч/дн/нед] назад» (серый)
- Тултип не нужен — информация показывается прямо в строке.
- Шрифт строки: 12px, `var(--text-muted)`.
- Отступ от имени: 2px.

### Форматирование «последний раз был активен»
- `< 1 мин` — «только что»
- `1-59 мин` — «был(а) X мин. назад»
- `1-23 ч` — «был(а) X ч. назад»
- `1-6 дн` — «был(а) X дн. назад»
- `7-30 дн` — «был(а) X нед. назад (с ДД.ММ.ГГГГ)»
- `> 30 дн` — «был(а) ДД.ММ.ГГГГ»

### Цвета
- `online` — зелёный (`#4caf50` или `var(--accent)`)
- `away` — жёлтый (`#ffc107` или `var(--text-muted)` с оттенком)
- `offline` — серый (`var(--text-muted)`)

## Тесты
- Сервер: `POST /api/me/activity` обновляет `last_active_at`, presence отдаёт `lastActiveAt`.
- Клиент e2e: эмуляция away-статуса ( time travel ), проверка отображения «был(а) X мин. назад» в ChatList и заголовке.

## Решение
### Сервер
- Миграция `0010_last_active_at.sql` — колонка `last_active_at` в `accounts`
- `touchActivity()` в `chat-helpers.ts` — обновление `last_active_at`
- `getLastActiveMap()` — получение `lastActiveAt` для списка пользователей
- `POST /api/me/activity` — пинг активности
- `GET /api/presence` — возвращает `{ presence: { [userId]: { online, away, lastActiveAt } } }`
- `loadChat` — возвращает `lastActiveAt` для участников
- `isAway` = online && `last_active_at` > 5 минут

### Клиент
- `reportActivity()` в `rest.ts`
- Пинг активности в `HomeScreen` — `mousemove`, `keydown`, `focus` с throttle 30с
- `AvatarBadges` — трёхцветный: online (зелёный), away (жёлтый), offline (серый)
- `formatLastSeen()` в `time.ts` — форматирование времени
- Subtitle в Conversation: online → «в сети», away → «отошёл. X мин. назад», offline → «был(а) X назад»
