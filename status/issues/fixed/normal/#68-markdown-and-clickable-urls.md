# #68 — Markdown в сообщениях и кликабельные ссылки

Реализовать поддержку форматирования текста в сообщениях (markdown) и сделать URL кликабельными.

## Текущее состояние
- `content.text` — чистый plain text
- `renderMentionText()` — заменяет только `@username` на `<span class="mention-highlight">`
- URL в тексте — просто текст, не кликабельный

## Поддерживаемый синтаксис

### Inline-форматирование
- `**жирный**` → `<strong>`
- `_курсив_` → `<em>`
- `` `код` `` → `<code>`
- `~~зачёркнутый~~` → `<del>`

### Ссылки
- Автоматический детект URL в тексте → `<a href="..." target="_blank" rel="noopener">`
- `[текст](url)` → явная ссылка с текстом
- URL'ы без markdown-разметки — тоже кликабельные

### Не поддерживать (оставить plain text)
- Блочный markdown (заголовки, списки, цитаты) — избыточно для мессенджера
- Вложенное форматирование — минимальный набор

## Реализация

### Парсер
- Не тащить библиотеку типа `marked` — свой lightweight-парсер
- Паттерн: regex + рекурсивная замена (как `renderMentionText`, но расширенный)
- Порядок: сначала inline-код (чтобы внутри кода не парсить остальное), потом bold/italic/strike, потом ссылки, потом @mentions
- XSS: не вставлять сырой HTML через `dangerouslySetInnerHTML`, а строить через React elements

### API
- Новая функция `renderMessageText(text, participants): React.ReactNode[]`
- Заменяет текущий вызов `renderMentionText()` в `Conversation.tsx`
- `renderMentionText` оставить как есть для обратной совместимости (или убрать, если вызов один)

### CSS
- `.message-link` — стиль для кликабельных ссылок (accent-цвет, underline при hover)
- Код — моноширинный фон как в Telegram

### Кликабельные URL
- Regex: `https?://[^\s<>\"']+` (или более строгий)
- Обрезать конечные знаки препинания (`,`, `.`, `)` и т.д.) если не часть URL
- `target="_blank"`, `rel="noopener noreferrer"`

## E2E тесты
- Текст с `**bold**` → рендерится как жирный
- Текст с `_italic_` → курсив
- Текст с `` `code` `` → моноширинный
- Текст с URL → кликабельная ссылка
- Текст с `[text](url)` → ссылка с текстом
- Внутри `` `code` `` markdown не парсится
- @mentions работают вместе с markdown

## Связь с другими задачами
- #66 (превью ссылок) — превью и кликабельные ссылки дополняют друг друга: превью для карточки, ссылка в тексте для доступа
- При наличии `kind:'link'` вложения — рендерить карточку; URL в тексте тоже кликабельный (не дублировать, а дополнять)
