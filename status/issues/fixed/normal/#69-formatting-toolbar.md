# #69 — Панель форматирования в композере (WYSIWYG)

Пользователям нужно удобно применять markdown-форматирование к тексту. Сделать панель инструментов / контекстное меню при выделении текста в композере. **Композер должен отображать отформатированный текст, а не управляющие коды.**

## Текущее состояние
- Markdown парсится при рендеринге (`renderMarkdown`)
- В композере нет визуальных инструментов форматирования
- Пользователь должен знать синтаксис `**bold**`, `_italic_` и т.д.
- Нет способа вставить ссылку `[текст](url)` кроме ручного набора
- Композер показывает сырой markdown (`**жирный**`) вместо отформатированного текста

## WYSIWYG-композер

### Как работает
- Композер отображает отформатированный текст в реальном времени
- `**жирный**` → отображается как **жирный**
- `_курсив_` → отображается как _курсив_
- `` `код` `` → отображается как `код`
- `~~зачёркнутый~~` → отображается как ~~зачёркнутый~~
- `[текст](url)` → отображается как ссылка
- При этом в `input` state хранится сырой markdown

### Реализация

#### Два слоя
1. **Background layer** — скрытый `<textarea>` с raw markdown (источник правды)
2. **Overlay layer** — `<div>` с отрендеренным markdown поверх textarea

#### Синхронизация
- Overlay точно повторяет размеры, padding, шрифт textarea
- Скролл textarea синхронизируется с overlay
- Caret позиция не видна в overlay (только в textarea)

#### CSS
- `.composer-raw` — textarea (прозрачный текст, виден caret)
- `.composer-rendered` — overlay с отрендеренным markdown
- Оба слоя абсолютно позиционированы, одинаковые размеры
- `pointer-events: none` на overlay (клики проходят в textarea)

#### Курсор и выделение
- Caret виден только в textarea (серый/чёрный)
- Overlay не перехватывает фокус и клики
- Выделение (selection) видно в textarea

## Панель форматирования

#### Расположение
- Над полем ввода (как в Telegram Desktop) — панель с кнопками
- Появляется **только при выделении текста** в композере
- Когда выделение снято — панель скрывается

#### Кнопки (слева направо):
1. **Bold** — `**` + выделенный текст + `**`
2. *Italic* — `_` + выделенный текст + `_`
3. ~~Strikethrough~~ — `~~` + выделенный текст + `~~`
4. `Code` — `` ` `` + выделенный текст + `` ` ``
5. 🔗 Link — диалог ввода URL, оборачивает выделенный текст в `[текст](url)`

#### Поведение
- При выделении текста в композере → панель появляется над полем ввода
- При снятии выделения → панель скрывается
- При клике на кнопку → обёртка выделенного текста markdown-разметкой
- Если текст не выделен (кнопка через горячие клавиши) → вставляет шаблон с плейсхолдером
- Курсор остаётся внутри композера

### Диалог ввода ссылки
- При нажатии кнопки 🔗 или `Ctrl+K`:
  - Если текст выделен → диалог "Вставьте URL" (текст уже есть)
  - Если текст не выделен → диалог "Введите текст и URL"
- Поля: "Текст ссылки" (prefilled выделенным текстом) и "URL"
- OK → `[текст](url)` вставляется в композер
- Cancel → ничего не происходит

### Горячие клавиши
- `Ctrl+B` → bold
- `Ctrl+I` → italic
- `Ctrl+K` → link (открывает диалог ввода URL)

## CSS
- `.formatting-bar` — панель над композером
- `.formatting-btn` — кнопка форматирования (иконка)
- `.formatting-btn.active` — активная кнопка (текст выделен этим форматом)
- `.formatting-btn:hover` — hover-эффект
- Разделитель `|` между группами кнопок
- `.composer-wrapper` — контейнер для textarea + overlay
- `.composer-raw` — textarea (прозрачный текст)
- `.composer-rendered` — overlay с markdown

## Интеграция с existing code
- Панель добавляется в `Conversation.tsx` над `.conv-input`
- Обработка выделения текста: `onSelect` на textarea
- Вставка текста: `inputRef.current.setRangeText()` + `inputRef.current.selectionStart/End`
- Overlay обновляется при каждом изменении input state

## Тесты
- E2E: ввод `**жирный**` → отображается как жирный в композере
- E2E: выделение текста → клик Bold → текст обёрнут в `**`
- E2E: клик Link → диалог → ввод URL → вставка `[текст](url)`
- E2E: `Ctrl+B` → bold
- E2E: отправка отформатированного сообщения → рендер как bold/italic/etc
