# #61 — Частые реакции в контекстном меню

Панель быстрых реакций в контекстном меню должна показывать сначала самые частые эмодзи, которые ставит текущий пользователь.

## Текущее состояние
- Панель реакций показывает фиксированный список: `👍 ❤️ 😂 😮 😢 🙏 🔥 👏`
- Порядок одинаковый для всех пользователей

## Ожидаемое поведение (как в Telegram Desktop)
- Первые 8 позиций — самые частые эмодзи-реакции текущего пользователя (отсортированы по убыванию количества использований)
- Если пользователь ставил меньше 8 уникальных реакций — оставшиеся позиции заполняются из дефолтного списка
- Счётчик обновляется при каждой новой реакции
- Данные хранятся на клиенте (localStorage), не на сервере

## Реализация (предварительная)
- **Хранение**: `localStorage` ключ `emoji_frequent` — объект `{ [emoji: string]: number }` (счётчик использований)
- **Обновление**: при вызове `toggleReaction` инкрементировать счётчик для выбранного эмодзи
- **Сортировка**: при открытии контекстного меню — взять топ-8 по счётчику, дополнить дефолтными
- **Компоненты**: изменить `ReactionBar` — вместо `QUICK_REACTIONS` использовать динамический список

## Приоритет
normal

## Решение
- `getFrequentReactions()` — читает `localStorage('emoji_frequent')`, сортирует по убыванию, возвращает топ-8, дополняет дефолтными
- `trackReaction(emoji)` — инкрементирует счётчик в `localStorage`
- `ReactionBar` — динамический список из `getFrequentReactions()`, `trackReaction` вызывается при каждом выборе
- Три точки входа: inline-реакция на сообщении, панель быстрых реакций, полный эмодзи-пикер
