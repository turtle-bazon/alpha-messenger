# #44 — Синхронизация прочитанных между устройствами

## Описание
Если прочитал сообщение на одном устройстве, на другом всё ещё показывает как непрочитанное (какое-то время или пока не обновится). Нужно чтобы read-статус синхронизировался сразу.

## Текущее поведение
- Read-событие отправляется на сервер (`POST /api/chats/:chatId/read`)
- Другие устройства узнают об этом только через WS `replay` или перезагрузку
- Нет механизм push-уведомления о read-событии между устройствами одного пользователя

## Решение
- Сервер при `chat.read` рассылает WS-событие `chat.read` всем другим подключениям того же пользователя
- Клиент обновляет `unreadCount` в списке чатов
