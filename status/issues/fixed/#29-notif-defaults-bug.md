# #29 — Некорректные дефолты уведомлений после очистки localStorage

[Баг] После удаления всех ключей localStorage и повторного входа — настройки уведомлений ведут себя странно. На скрине видно: «Звук» включён, «Уведомления браузера» выключены с подсказкой «Уведомления заблокированы в настройках браузера», но ключей `alpha.notif.sound` и `alpha.notif.browser` в localStorage вообще нет (только auth-ключи).

Проблемы:
1. Дефолтное поведение при отсутствии ключей: `readFlag()` считает отсутствие = `true` (включено). Но в UI показывается что браузерные выключены. Непонятно почему — возможно `ensureBrowserPermission()` при входе определяет `denied` и молча переключает, но не пишет в localStorage.
2. Если браузер заблокировал уведомления (`Notification.permission === 'denied'`), `ensureBrowserPermission()` не сохраняет это в localStorage. Каждый новый логин = повторная попытка запроса (бессмысленная).
3. Ключи `alpha.notif.sound` / `alpha.notif.browser` не появляются в localStorage при первом входе (дефолт не сидит явно).

Клоду: проверить весь пайплайн дефолтов уведомлений. При входе: если ключей нет — явно записать дефолты (`'1'` для обоих). Если `Notification.permission === 'denied'` — записать `'0'` для browser и больше не спрашивать. Убедиться что UI корректно отражает состояние.
