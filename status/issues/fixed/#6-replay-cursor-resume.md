# #6 — Resume потока по сохранённому курсору lastSeq

Reconnect/reload реплеил всю историю outbox с нуля и рисковал выдать её за live (уведомления/звуки/бейджи). Причина: клиент не сохранял курсор потока между сессиями — при создании WsClient lastSeq=0, и сервер реплеил всё. Решение: lastSeq сохраняется в localStorage per-account (session.ts: getLastSeq/setLastSeq, чистится в clearSession), WsClient зовёт onSeqAdvance при продвижении курсора, HomeScreen сидит WsClient сохранённым lastSeq и пишет его обратно — после reload реплеится только пропущенное. Граница replay/live (isLive по 'synced') уже отсекала историю от живых событий — это сохранено. Покрыто e2e reconnect.spec.ts (после reload security-уведомление повторно не всплывает, lastSeq в localStorage не сбрасывается).
