# #1 — Единый префикс /api/ для REST

Все серверные REST-эндпоинты объединены под общим префиксом /api/ (app.ts: api-роуты регистрируются плагином с prefix:'/api', /ws остаётся в корне). Поправлены клиент (apiUrl добавляет /api — единый источник префикса), серверные тесты, e2e-хелпер и doc/api.md. Apache-прокси упрощён: одно правило ProxyPass /api/ вместо семи. Проверено: серверные тесты 8/8, e2e-регресс 9/9.
