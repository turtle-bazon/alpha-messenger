# #3 — Статус ✓✓ переживает переоткрытие чата

Статус прочтения ✓✓ деградировал в ✓ при повторном открытии чата (или переподключении WS). Причина: позиция прочтения собеседником (readUpTo) жила только в эфемерном состоянии клиента и сеялась лишь из live-события message.read — при перемонтировании Conversation сбрасывалась в 0, а старое событие повторно не реплеилось (resume отдаёт только события после lastSeq). Сервер хранит last_read_message_id, но не отдавал чужую позицию. Решение: loadChat отдаёт peerReadUpTo (max last_read_message_id среди других участников), Conversation сидит им readUpTo при открытии, HomeScreen держит peerReadUpTo в объекте чата актуальным по live message.read (даже у закрытых чатов). Покрыто e2e interactions.spec.ts («статус ✓✓ переживает переоткрытие чата»).
