# #58 — Мультиязычность (i18n)

Добавить поддержку нескольких языков в интерфейсе клиента.

## Текущее состояние
Все строки интерфейса захардкожены на русском языке. Нет системы локализации.

## Поведение
- Переключение языка в настройках
- Автоопределение языка браузера при первом запуске
- Сохранение выбора в localStorage
- Загрузка строк по требованию (не все языки сразу)

## RTL (right-to-left)
Для арабского, иврита, фарси и т.д.需要:
- Направление текста `dir="rtl"` на корневом элементе
- Зеркальный CSS (flip) для layout: sidebar справа, стрелки влево, выравнивание
- `flex-direction: row-reverse` / `margin-left` ↔ `margin-right` через CSS-переменные или logical properties
- Чат-список и сообщения: отправитель справа, получатель слева (как в Telegram)
- Текстовые поля: выравнивание по правому краю
- Иконки с направлением (стрелки, навигация) — зеркалить
- Не зеркалить иконки без направления (сердце, звезда и т.д.)

## Реализация
- Библиотека: i18next + react-i18next (или аналог)
- Структура: `locales/{ru,en,ar,...}/translation.json`
- Хук `useTranslation()` для доступа к строкам
- Постепенная миграция строк (не всё сразу)
- CSS: использовать logical properties (`margin-inline-start` вместо `margin-left`)
- Тестировать с реальными RTL-языками
