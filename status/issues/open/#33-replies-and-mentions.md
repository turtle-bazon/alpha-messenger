# #33 — Ответы на сообщения и навигация по упоминаниям (@)

Реализовать систему ответов на сообщения и навигацию по упоминаниям/ответам вам, как в Telegram.

## Серверная часть

### Ответ на сообщение
- При отправке сообщения можно указать `replyToMessageId` — ID сообщения, на которое отвечаем.
- `replyToMessageId` сохраняется в таблице `messages` (новая колонка, nullable FK на messages.message_id).
- `loadMessages` отдаёт `replyToMessageId` в ответе.
- При редактировании/удалении — `replyToMessageId` не меняется.

### Уведомления об ответах
- Когда кто-то отвечает на ваше сообщение — это **отдельное уведомление** (не просто новое сообщение).
- Звук для ответов — тот же что и для обычных уведомлений (настройка `alpha.notif.sound`).
- В WebSocket-событии `message.new` добавить флаг `isReply: boolean` и `replyToMessageId`.
- На клиенте: если `isReply && senderId !== myId` — триггерить уведомление с текстом «Ответил на ваше сообщение» + превью текста.

### Счётчик упоминаний
- В `loadChat` добавить поле `unreadMentions` — количество непрочитанных сообщений, которые отвечают на ваши сообщения или содержат @ваш_username.
- Сервер определяет: сообщение является «ответом вам» если `replyToMessageId` указывает на сообщение автора текущего пользователя.

## Клиентская часть

### Инициирование ответа
- При наведении на сообщение (hover) — справа от пузыря появляется иконка-стрелка «↩» (reply, как в Telegram).
- Клик по стрелке — активируется режим ответа: над композером появляется плашка «Ответ на: <аватар> <имя> <текст>», фокус переходит в поле ввода.
- Режим ответа отменяется по: клику × на плашке, отправке сообщения, нажатию Escape.
- Второй клик по стрелке на том же сообщении — отмена ответа (toggle).
- На мобильных (≤600px): свайп вправо по сообщению — как альтернативный жест для ответа (как в Telegram).

### Индикатор @ в ChatList
- В списке чатов рядом с непрочитанными показывать кнопку/иконку «@» (как в Telegram).
- Если `unreadMentions > 0` — показать бейдж с числом поверх или рядом с чатом.
- Клик по «@» — открывает чат и прокручивает к первому непрочитанному ответу вам.

### Навигация по ответам в Conversation
- В области переписки (справа внизу, рядом с композером) — кнопка «@» (или стрелка вниз с бейджем).
- При нажатии — прокрутка к следующему непрочитанному сообщению, отвечающему на вас.
- Циклическая навигация: достигнув последнего — возврат к первому.
- Кнопка видна только если есть непрочитанные ответы вам в текущем чате.

### Визуальное выделение
- Сообщения, отвечающие на вас, выделяются слева полоской (как в Telegram — цветная вертикальная линия у края пузыря).
- При скролле к такому сообщению — плавная анимация подсветки.

### Reply UI в композере
- При ответе на сообщение — над полем ввода появляется плашка «Ответ на: <текст>» с кнопкой закрытия (×).
- Плашка показывает превью сообщения, на которое отвечаем (аватар + имя + первые ~50 символов текста).
- Клик по плашке — прокрутка к оригинальному сообщению.
- × — отмена ответа (плашка исчезает).

### Определение «ответ на вас»
- Сообщение считается «ответом на вас» если:
  - `replyToMessageId` указывает на сообщение, автором которого является текущий пользователь, ИЛИ
  - Текст содержит @username текущего пользователя (в будущем, пока — только replyToMessageId).

## Тесты
- Сервер: отправка с replyToMessageId, loadMessages отдаёт replyToMessageId, unreadMentions корректно считается.
- Клиент e2e: два клиента, один отвечает на сообщение другого — у получателя появляется бейдж @, клик прокручивает к ответу, уведомление срабатывает.
