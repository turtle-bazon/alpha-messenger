# #77 — Управление клиентскими сессиями

## Описание
Нужна возможность просматривать и удалять старые клиентские сессии (устройства).

## Требования

### Функциональные
- Список активных сессий пользователя (устройства, дата входа, IP)
- Удаление отдельных сессий
- Удаление всех сессий кроме текущей

### Безопасность
- При входе с нового device ID должна быть задержка перед получением доступа к sensitive данным (например, история сообщений за долгий период)
- Возможность настроить "доверенные устройства"
- Уведомление пользователю при входе с нового устройства
- Ограничение на максимальное количество активных сессий

## Приоритет
Средний — важная фича для безопасности, но не критичная для базового функционала.
