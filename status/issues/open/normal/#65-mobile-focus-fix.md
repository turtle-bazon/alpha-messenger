# #65 — Исправить фокус: только при попытке ввода

## Описание
Текущая логика фокуса слишком агрессивна — фокусирует поле ввода при открытии чата, клике в пустую область и т.д. На мобильных это вызывает открытие клавиатуры и мигание. На десктопе тоже нет смысла — пока пользователь не начал набирать, поле ввода не нужно.

### Поведение (одинаково на всех платформах)

**Фокус в поле ввода сообщения** — ТОЛЬКО когда:
- Пользователь нажимает printable клавишу на клавиатуре (без модификаторов)
- Пользователь делает paste
- Пользователь кликает напрямую на textarea
- Пользователь выбирает эмодзи/стикер/упоминание → возврат фокуса
- Пользователь закрывает модал Escape → возврат фокуса

**НЕ фокусировать** при:
- Открытии чата (`useEffect` по `[chatId]`) — убрать
- Клике в пустую область scroll area — убрать
- Клике в пустую область chat list — убрать

**Умная фокусировка при вводе с клавиатуры:**
Если пользователь нажимает клавишу и фокус НЕ на поле ввода:
- Если открыт модал с полем ввода (NewChatDialog, MembersDialog search) → фокус туда
- Если фокус на списке чатов → фокус в поле поиска чатов
- Иначе → фокус в поле ввода сообщения

**Фильтрация модалей:** `.backdrop`, `[data-testid="emoji-picker"]`, `[data-testid="context-menu"]` — при их наличии фокус не смещаем (оставляем в модале).
