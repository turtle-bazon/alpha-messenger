# #33 — Ответы на сообщения и навигация по упоминаниям (@)

Реализовать систему ответов на сообщения и навигацию по упоминаниям/ответам вам, как в Telegram.

## Реализовано ✅

### Серверная часть
- Миграция `0006_reply_to_message.sql` — колонка `reply_to_message_id` (nullable FK)
- `POST /chats/:id/messages` — принимает `replyToMessageId`, валидирует существование в том же чате
- `GET /chats/:id/messages` — отдаёт `replyToMessageId` в каждом сообщении
- `WS message.new` — поля `replyToMessageId` и `isReply` в payload
- `loadChat` — поле `unreadMentions` (непрочитанные ответы на мои сообщения)

### Клиентская часть
- Типы `Message`, `Chat`, `SendResult` обновлены
- `sendMessage` принимает `replyToMessageId`
- Кнопка ↩ при hover на сообщении (для всех, не только своих)
- Reply preview в пузыре (голубая полоска слева + имя + текст, клик прокручивает к оригиналу)
- Плашка ответа над композером (имя + текст + × для отмены)
- Escape отменяет ответ
- CSS стили: `.bubble-reply-btn`, `.bubble-reply`, `.conv-reply-banner`

## Осталось (не критично)
- Уведомление об ответе (отдельное от обычного message.new) — пока не реализовано
- Индикатор @ в ChatList (бейдж unreadMentions) — сервер готов, клиент не тронут
- Навигация по @ в Conversation (кнопка вниз с бейджем) — не реализовано
- Свайп вправо для ответа на мобильных — не реализовано
- Подсветка сообщений, отвечающих на вас (цветная полоска слева) — не реализовано
