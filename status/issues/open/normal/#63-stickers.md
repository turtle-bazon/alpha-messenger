# #63 — Стикеры (как в Telegram)

## Описание
Полноценная поддержка стикеров по аналогии с Telegram:

- Набор стикеров (паки) — пользователь может добавлять/удалять паки
- Каждый пак содержит набор стикеров (1–140 шт.)
- Отправка стикера как отдельного сообщения (не внутри текста)
- Стикеры отображаются крупно, без облачка (как в Telegram)
- Миниатюра пака — первый стикер пака, или обложка

## UI (вкладки в композере)
Иконка смайлика в поле ввода открывает нижнюю панель с вкладками (как в Telegram):
- **Эмодзи** — текущий EmojiPicker
- **Стикеры** — выбор из установленных паков, часто используемые сверху
- **GIF** — опционально, в v1 можно отложить

## Реализация (v1)
- Формат: WebP (статические) или PNG (с прозрачностью)
- Хранение: бинарные blob через существующий механизм `/api/blobs`
- Метаданные паков и стикеров: новая таблица `sticker_packs` + `sticker_items`
- API: CRUD для паков, загрузка/скачивание стикеров
- Анимированные стикеры (Lottie) — опционально, в v1 не обязательны
- Стикеры на прозрачном фоне (WebP/PNG с альфа-каналом)
- Отправка: стикер как сообщение с типом `sticker` (вместо `text`)
