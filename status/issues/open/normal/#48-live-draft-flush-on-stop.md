# #48 — Live Draft: допослать незавершённый текст перед скрытием

## Описание
При Live Draft (#18): если пользователь набирал текст, остановился и прекратил набор — незавершённый текст теряется. Нужно, чтобы при остановке набора последняя версия draft была отправлена ещё раз (с debounce), и только после таймаута без нового ввода — draft исчезал.

## Текущее поведение
1. Пользователь печатает → отправляется typing с draft
2. Останавливается → через 6 сек таймаут draft исчезает
3. Последний введённый текст мог быть отправлен 6+ секунд назад, а текущий — нет

## Ожидаемое поведение
1. Пользователь печатает → draft отправляется с debounce ~300мс
2. Останавливается → через ~1 сек без ввода отправляется финальный draft
3. Через 6 сек без нового ввода → draft исчезает

## Реализация
- В `onInputChange`: при остановке ввода (debounce ~1 сек) допослать текущий draft
- Или: `sendTyping` с текущим текстом при потере фокуса полем ввода
