# #36 — Статус «отошёл» и «последний раз был активен»

Реализовать статус «отошёл» для пользователей, которые залогинены, но давно не проявляли активность в мессенджере.

## Серверная часть

### Отслеживание активности
- Новая колонка в `accounts`: `last_active_at` (timestamptz, nullable). Обновляется при любой активности пользователя.
- Обновление `last_active_at`:
  - При отправке сообщения (POST /api/messages) — уже есть.
  - При WS-соединении / reconnect — обновлять при hello.
  - Новый REST-эндпоинт: `POST /api/me/activity` — «пинг» активности. Клиент шлёт при обнаружении движения мыши, кликах, нажатиях клавиш (throttle ~30 сек).
- В `loadChat` и `GET /api/presence` добавить `lastActiveAt` для каждого участника.

### Определение статуса
- `online` — есть живой WS-connection (как сейчас).
- `away` — `last_active_at` > 5 минут назад, но WS есть.
- `offline` — нет WS-connection.

## Клиентская часть

### Отправка пингов активности
- В `HomeScreen` или `App` — обработчик `mousemove`, `keydown`, `focus` с throttle 30 сек.
- При срабатывании — `POST /api/me/activity` (fire-and-forget, без ожидания ответа).
- Не шлём пинг если вкладка не активна (`document.hidden`).

### Отображение статуса в ChatList
- Под именем чата (в двухстрочном элементе) — строка статуса:
  - `online` — «в сети» (зелёный, как сейчас).
  - `away` — «отошёл» (жёлтый/оранжевый, `var(--text-muted)` или отдельный цвет).
  - `offline` — «был(а) недавно» / «был(а) X [минут/часов/дней] назад».
- Для DM: показывать статус собеседника.
- Для групп: не показывать (или показывать количество участников в сети, как сейчас).

### Отображение в заголовке переписки
- Под именем собеседника (DM) — та же логика статуса.
- Для групп: «N участников, M в сети» (без change).

### Форматирование «последний раз был активен»
- `< 1 мин` — «только что»
- `1-59 мин` — «был(а) X мин. назад»
- `1-23 ч` — «был(а) X ч. назад»
- `1-6 дн` — «был(а) X дн. назад»
- `7-30 дн` — «был(а) X нед. назад (с ДД.ММ.ГГГГ)»
- `> 30 дн` — «был(а) ДД.ММ.ГГГГ»

### Цвета
- `online` — зелёный (`#4caf50` или `var(--accent)`)
- `away` — жёлтый (`#ffc107` или `var(--text-muted)` с оттенком)
- `offline` — серый (`var(--text-muted)`)

## Тесты
- Сервер: `POST /api/me/activity` обновляет `last_active_at`, presence отдаёт `lastActiveAt`.
- Клиент e2e: эмуляция away-статуса ( time travel ), проверка отображения «был(а) X мин. назад» в ChatList и заголовке.
