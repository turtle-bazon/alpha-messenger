# #32 — Превью ссылок (unfurl)

Карточки-превью для ссылок в сообщениях: заголовок, описание, имя сайта,
картинка — как в desktop Telegram. Живой предпросмотр в композере по мере набора
URL, с возможностью снять превью крестиком и отправить голую ссылку.

## Архитектура (с учётом E2EE)
Сервер не читает ciphertext (не развернёт ссылку из сообщения), а браузер не
может фетчить произвольные кросс-доменные страницы (CORS) — поэтому:

* Серверный хелпер **`POST /api/unfurl {url}`**: сервер сам тянет страницу с
  **SSRF-защитой** (только http/https; блок приватных/loopback/link-local
  адресов; лимит редиректов с перепроверкой каждого хопа; таймаут; потолок
  размера HTML; только `text/html`), парсит OpenGraph/`<title>`/`meta
  description` и возвращает `{ url, title, description, siteName, image? }`.
* OG-картинку сервер тоже скачивает (SSRF-guard + лимит размера) и возвращает её
  **байтами** (base64) — чтобы клиент мог нарисовать её в canvas без CORS-taint
  (бэкенд — свой origin) и ужать в маленький inline-thumbnail. Сервер не тащит
  image-библиотеки, ресайз — на клиенте.
* Клиент отправителя вшивает превью в сообщение как вложение **`kind:'link'`**
  (в ciphertext, по модели `attachments[]` из #31). Получатель просто рендерит
  карточку — без повторного фетча, не светит свой IP стороннему сайту.

`LinkAttachment { kind:'link', url, title, description, siteName, thumb }` —
`thumb` маленький inline-JPEG (как у картинок), без блоба (lightbox не нужен,
клик ведёт на сайт). Разворачиваем **первую** ссылку в сообщении.

## UX
Живой предпросмотр в композере (debounce): печатаешь URL → над полем ввода
карточка, её можно снять крестиком (запоминаем снятый URL, чтобы не всплывал
снова) и отправить без превью. При отправке готовое превью уезжает вложением.

## Приватность (осознанная цена)
Раз сервер разворачивает ссылку — он видит URL'ы, которыми делишься (текст
сообщений по-прежнему не видит). Без серверного фетча превью в браузере не
сделать в принципе. Смягчение: крестик на превью; на вырост — глобальный тумблер.

## SSRF (обязательно)
* Только схемы http/https.
* Резолв хоста и отказ на приватные/loopback/link-local/ULA диапазоны (в т.ч.
  на каждом редиректе — защита от DNS-rebind через цепочку редиректов).
* Лимит числа редиректов, таймаут запроса, потолок вычитываемого размера.
* В dev/тестах фетч localhost разрешается флагом `UNFURL_ALLOW_PRIVATE=1`
  (по умолчанию выключен).
