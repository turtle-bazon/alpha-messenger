# #73 — Уведомления о реакциях

## Описание
Пользователь никак не узнаёт, что на его сообщение поставили реакцию. Может только догадаться, увидев значок в UI. Нужно уведомлять об этом — push-уведомлением (на мобильном/десктопе) и/или внутри приложения.

## Ожидаемое поведение
- Когда кто-то ставит реакцию на сообщение пользователя — приходит уведомление
- На десктопе — системное уведомление (если окно не в фокусе) + индикатор в UI
- На мобильном — push-уведомление
- В داخلнем списке уведомлений (если появится) — пункт о реакции
- Не уведомлять о собственных реакциях

## Текущее поведение
Реакция отображается в UI (значок под сообщением), но push/internal уведомлений нет.

## Контекст
- Реакции уже реализованы (#23)
- Push-уведомления работают для сообщений (FCM/UnifiedPush)
- Сервер уже отправляет WS-событие `reaction.added` — нужно только добавить обработку на клиенте и, возможно, серверную часть для push
