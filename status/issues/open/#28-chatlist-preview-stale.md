# #28 — Баг: превью последнего сообщения в ChatList не обновляется

[Баг] Превью последнего сообщения в ChatList не обновляется при получении новых сообщений через WebSocket. Наблюдение: в ChatList у чата отображается устаревшее превью с old временем, хотя в самом чате уже видны более новые сообщения. Это значит, что поле lastMessage в объекте чата не обновляется при инкрементальном обновлении через WS.

Клоду: проверить весь пайплайн — как WS-событие message приходит на клиент, как обновляется состояние чатов (chats state), как ChatList читает lastMessage из этого состояния, и где именно происходит (или не происходит) обновление lastMessage при новом сообщении. Возможно lastMessage обновляется только при полной перезагрузке списка чатов (GET /chats), но не при инкрементальном WS-обновлении.
