1. Дедупликация direct-чата не защищена от гонки: два одновременных POST /chats (direct) с теми же участниками могут создать два чата. Для v1 приемлемо. Решение при необходимости — уникальный ключ по канонической паре участников (least/greatest user_id) или advisory-lock на пару.
2. Поддержка вставки изображений из буфера обмена (Ctrl+V / Cmd+V) в поле ввода сообщения в Conversation. При вставке картинки — открывать ImageEditor (как при прикреплении через 📎) для предпросмотра, поворота, подписи и отправки.
3. Live Draft — ультима-фича: реалтайм-превью набираемого сообщения. Когда пользователь начинает набирать текст в поле ввода, собеседник (или участники группы) видит его облачко в реальном времени — буквы появляются по мере набора. Уведомления и звуки НЕ срабатывают до фактической отправки. Детали:

**Клиент-отправитель:**
- При вводе текста (debounce ~300ms) отправлять WS-событие `typing` с текущим `draft` (полный текст, не только сигнал «печатает»)
- При очистке поля или потере фокуса — отправить `typing` с пустым draft (означает «прекратил набор»)
- При отправке сообщения — draft автоматически удаляется

**Клиент-получатель:**
- По событию `typing` с непустым draft — показать облачко-превью в области переписки (внизу, как обычное сообщение, но визуально отличное)
- Облачко-превью обновляется в реальном времени по мере получения новых `typing`-событий
- Когда пользователь прекращает набор (typing с пустым draft или таймаут ~10 сек) — облачко исчезает с анимацией fade-out
- Облачко-превью НЕ вызывает уведомления (нет звука, нет badge, нет push) — только визуальное обновление

**Визуальное отличие от отправленного сообщения:**
- Пунктирная рамка вместо сплошной
- Полупрозрачный фон (opacity 0.6-0.7)
- Подпись «печатает...» вместо времени
- Анимация появления (fade-in + slide-up)
- Анимация исчезновения (fade-out + slide-down)

**Сервер:**
- Расширить WS-событие `typing`: добавить поле `draft` (string, опционально). При отсутствии draft — классический сигнал «печатает» (обратная совместимость)
- Хранить draft не нужно — только эфемерная передача через WS (не в outbox, не в БД)
- Группы: draft-облачко видно всем участникам, кроме автора

**Ограничения:**
- Draft передаётся как есть (без шифрования) — аналогично typing,.acceptable для v1
- Максимальная длина draft — 1000 символов (дальше не показывать, только «печатает...»)
- Задержка отображения — не более 500ms от набора до появления у получателя
4. Кнопка отправки (самолётик) ведёт себя неудобно: когда поле пустое — кнопки нет, справа пустое место; когда начинаешь писать — кнопка появляется. Визуально «прыгает». Нужно: кнопка всегда видна, но в неактивном состоянии (приглушённый цвет, disabled), когда поле пустое. При вводе текста — становится активной (яркий цвет, кликабельная). Без анимации появления/исчезновения — стабильная позиция всегда.
5. Версионирование сообщений (правки и удаления). При редактировании или удалении сообщения предыдущая версия сохраняется. Доступ к истории — только для «админов» чата: в группе это создатель, в личном чате — оба участника. Обычные участники группы видят пометку «ред.» на отредактированных, удалённые сообщения — не видят вообще (скрыты). Админ может посмотреть полную историю изменений/удалений любого сообщения в чате.
6. Идентификация отправителя в групповом чате. Сейчас в группе не видно кто прислал сообщение — все пузыри одинаковые. Нужно: аватар отправителя слева от пузыря (как в Telegram), имя/username над сообщением (для первого сообщения от данного автора в серии подряд). В личном чате аватар и имя отправителя не показываются — незачем.
7. Профили пользователей. Каждый пользователь имеет профиль, который можно открыть (клик по аватару/имени в чате, участнику в списке участников и т.д.). Профиль содержит базовую информацию: username, дату регистрации, последний вход. Помимо этого — каждый пользователь может вести личные пометки о других пользователях (заметки видны только самому автору). Например: «это Петя из отдела маркетинга», «договориться о встрече», «не давать доступ к проекту X». Заметки привязаны к паре (myUserId, targetUserId), редактируемые, с историей изменений.
8. Смайлики. Два аспекта: (а) вставка эмодзи в текст сообщения — пикер/панель эмодзи в композере, открытие по кнопке 😊 или по клику, поиск по названию, недавние; (б) реакции на сообщения — возможность добавить эмодзи-реакцию к любому сообщению (как в Telegram/Discord), под сообщением отображаются收集анные реакции с счётчиками, клик по реакции добавляет/убирает свою. Реакции — live через WS, хранятся на сервере.
9. Галочки прочтения (✓✓) — сделать посимпатичнее. Сейчас выглядят как простые текстовые символы Unicode. Ориентир — Telegram: маленькие (12-14px), аккуратные, с правильным отступом от времени. Реализовать через inline SVG (двойная галочка как путь path), а не через текстовые символы. Два размера галочки: одна (отправлено) и двойная (доставлено/прочитано). Три статуса: отправлено (одна серая), доставлено (две серые), прочитано (две синие/зелёные). Отступ между временем и галочками — 2-4px. Сама галочка — 14x14px, stroke-width ~1.5, без заливки (только контур).
11. [Баг] Выпадающее меню настроек уведомлений (колокольчик в шапке) — обрезается слева, текст нечитаем («Звук» → «вук», «Уведомления браузера» → «ведомления браузера»). Меню слишком узкое и/или позиционируется неправильно относительно кнопки. Нужно: min-width足够的 для самого длинного текста, выравнивание по правому краю кнопки (как в Telegram — меню открыто вниз влево от колокольчика), без обрезки.
12. Поле ввода сообщения должно расширяться по мере ввода текста. Сейчас при многострочном тексте окно скроллится и предыдущий текст не виден. Нужно: textarea растёт в высоту по мере добавления строк (макс. высота ~150-200px, после — внутренний скролл), отправка по Enter (Shift+Enter — перенос строки). Рост плавный, без «прыжков».
13. Автоматический повтор неотправленных сообщений. Если отправка не удалась (сеть упала, сервер недоступен) — сообщение помечается как failed и показывается пользователю. При восстановлении связи (или по таймеру) — автоматическая повторная отправка. Порядок строго последовательный: первое неотправленное идёт первым, второе — вторым. Нельзя допустить чтобы более позднее сообщение обогнало раннее. Кнопка «повторить» рядом с failed-сообщением для ручного повтора. При автоматическом повторе — визуальный индикатор (спиннер или анимация).
14. Настройки уведомлений: галочка по умолчанию ВКЛЮЧЕНА. При загрузке сайта: если галочка включена — запрашивать разрешение браузера на уведомления (Notification.requestPermission()). Если галочка выключена — не запрашивать. При первом входе в учётную запись (или обновлении) — если галочка включена, сразу запрашивать разрешение. Повторно не запрашивать если разрешение уже given/denied. Логика: сначала проверить localStorage (есть ли сохранённая настройка), если нет — дефолт = включена + запросить разрешение.
15. Статусы онлайн/оффлайн/тайпинг в ChatList как в Telegram. Реализовать три состояния аватара:
- **Онлайн**: зелёный кружок внизу справа аватара (12-14px). Только для DM.
- **Оффлайн**: серый кружок внизу справа аватара. Только для DM.
- **Тайпинг**: большой круг-окантовка вокруг всего аватара (2-3px border). Цвет — контрастный с аватаром (пусть клод подберёт подходящий). Появляется: в DM — если собеседник печатает тебе; в групповых чатах — не показывать (или показывать в списке участников при необходимости).
- **Групповые чаты**: кружок статуса НЕ показывать. Тайпинг-окантовку тоже НЕ показывать.

Компонент: позиционирование bottom:0 right:0 для кружков, border-radius:50% для тайпинга. Не путать с AvatarStatus (оранжевый «typing» кружок) — это два разных индикатора.
