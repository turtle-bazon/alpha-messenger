1. Дедупликация direct-чата не защищена от гонки: два одновременных POST /chats (direct) с теми же участниками могут создать два чата. Для v1 приемлемо. Решение при необходимости — уникальный ключ по канонической паре участников (least/greatest user_id) или advisory-lock на пару.
2. В run/deploy/apache-proxy.conf не хватает LoadModule строк (proxy, proxy_http, proxy_wstunnel, headers). Записать в документацию (doc/deploy.md или аналог), что при установке Apache нужно включить эти модули, но в сам конфиг их не добавлять — они зависят от дистрибутива. Также в конфиге вместо localhost используется <app-host> как плейсхолдер — при деплое подставить реальный адрес сервера.
3. На экранах логина и регистрации (LoginScreen, RegisterScreen) добавить глазик рядом с полем пароля для показа/скрытия введённого текста. Тип input чередовать password/text по клику на иконку.
4. При создании группы (NewChatDialog, вкладка «Группа»): (а) вместо свободного ввода username для добавления участников — поиск по списку пользователей, с кем уже есть чаты (GET /chats, собрать уникальных участников direct-чатов); (б) нельзя создать группу без участников, но создатель автоматически входит в группу и не показывается в списке добавляемых (сервер уже добавляет его при POST /chats type:group).
