1. Дедупликация direct-чата не защищена от гонки: два одновременных POST /chats (direct) с теми же участниками могут создать два чата. Для v1 приемлемо. Решение при необходимости — уникальный ключ по канонической паре участников (least/greatest user_id) или advisory-lock на пару.
2. Клиент при деплое за Apache-прокси ходит напрямую в localhost:3000 вместо домена. Причина: VITE_API_URL захардкожен в http://localhost:3000 при сборке. На скриншоте (screen.png) видно CORS-ошибку и запросы к localhost:3000/register. Исправление: в run/prod/docker-compose.yaml при сборке клиента прокинуть VITE_API_URL=https://doyhof3v5x.java-lords.ru как build arg, пересобрать и запушить образ в ghcr.io. Это касается только run/prod — там собираются и публикуются образы.
3. Все серверные API-эндпоинты (/auth/*, /me, /chats/*, /messages/*, /devices, /push/*, /health) разбросаны по корню. Нужно объединить под общим префиксом /api/ (например /api/auth/*, /api/me, /api/chats/*). Это упростит Apache-прокси: один ProxyPass /api/ http://localhost:3000/api/ вместо семи отдельных правил. На клиентах (REST-клиент, e2e-тесты) поправить базовый URL. Задача для вечернего фикса.
